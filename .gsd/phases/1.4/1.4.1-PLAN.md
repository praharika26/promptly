# Plan 1.4.1: Cloud Convex Project & Schema Design

## Objective
Initialize a cloud-hosted Convex Database project, establish the schema for Promptly (agents, users, transactions), and wire up the basic Next.js Convex providers so we can transition from static mocks to a live backend.

## Context
- .gsd/SPEC.md
- .gsd/ARCHITECTURE.md
- We are moving away from Supabase to Convex DB on the cloud.
- The `npx convex login` should be completed before proceeding.

## Tasks

<task type="auto">
  <name>Initialize Convex & Install Dependencies</name>
  <files>
    - package.json
  </files>
  <action>
    - Install `convex` package (`pnpm add convex`).
    - Note: Assuming `npx convex login` is done, run `npx convex dev` which prompts to create a new **cloud project**. This provisions a real cloud DB on Convex's managed infrastructure.
    - Since this step requires user interaction to select project names on the cloud, we will run it as a verification step, or I can run the non-interactive setup if a project ID is specified.
    - We will also add `@convex-dev/react` and `convex` to the dependencies.
  </action>
  <verify>Check that `convex.json` exists and dependencies are in `package.json`</verify>
  <done>Convex is installed and initialized successfully.</done>
</task>

<task type="auto">
  <name>Create Convex Cloud Schema</name>
  <files>
    - convex/schema.ts
  </files>
  <action>
    - Create `convex/schema.ts`.
    - Design the schema for the AI Agents marketplace:
      - `users`: `walletAddress` (string), `name` (string), `reputation` (number), `earnings` (number), `createdAt` (number).
      - `agents`: `id` (v.id("agents")), `name` (string), `description` (string), `imageUrl` (string), `price` (number), `creatorWallet` (string), `rating` (number), `status` (string).
      - `transactions`: `agentId` (v.id("agents")), `buyerWallet` (string), `amount` (number), `timestamp` (number).
    - Use `defineSchema` and `defineTable` from `convex/server`.
    - Ensure indexes are added (e.g., search agents by creator, or users by walletAddress).
  </action>
  <verify>Verify `convex/schema.ts` has no TypeScript errors and exports the schema.</verify>
  <done>Schema accurately reflects the Promptly app domain model.</done>
</task>

<task type="auto">
  <name>Setup Convex Provider in Next.js</name>
  <files>
    - app/ConvexClientProvider.tsx
    - app/layout.tsx
  </files>
  <action>
    - Create a new client component `ConvexClientProvider.tsx` wrapping the application in `<ConvexProvider client={convex}>`.
    - Initialize the ConvexReactClient with `process.env.NEXT_PUBLIC_CONVEX_URL`.
    - Update `app/layout.tsx` to include the `ConvexClientProvider` inside the `html/body` structure, alongside or wrapping any existing context providers.
  </action>
  <verify>Running the app (`npm run dev`) does not throw context or provider errors.</verify>
  <done>The React tree is properly wrapped with Convex context.</done>
</task>
